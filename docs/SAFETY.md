# TinyAgent safety model

TinyAgent is a local coding agent. The model runs locally through TinyEngine, but the agent can use tools that affect your machine.

## Tool modes

- Default mode asks before write/exec tools.
- `--yes` auto-approves tools for the session.
- `--dry-run` prevents write/exec tools from running.
- `--no-network` disables network tools.
- `--ask` answers conversationally and never uses tools.

## Recommended usage

Use the safest mode that fits the task:

```bash
bin/tinyagent --ask "Explain this repository"
bin/tinyagent --dry-run "Plan the changes needed for issue 123"
bin/tinyagent --no-network "Refactor this parser and run tests"
```

Only use `--yes` in repositories and prompts you trust.

## Data handling

Model inference is local. Tool calls may still read files, write files, run commands, or access network resources depending on approval mode and flags. Review session files and generated diffs before sharing logs or benchmark outputs.

## Untrusted inputs

Treat these as untrusted:

- Prompt text from other people.
- Repositories you have not reviewed.
- Shell commands generated by a model.
- GGUF files from unknown sources.
- Benchmark artifacts containing command output or local paths.
